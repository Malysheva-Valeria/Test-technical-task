# IT Asset Management Module для Odoo 17

Модуль для управління IT-активами компанії з підтримкою системи заявок та порталом для співробітників.

## Опис

Цей модуль створений як тестове завдання для компанії UBM Soft і надає комплексне рішення для управління IT-активами організації. Дозволяє відстежувати обладнання, керувати життєвим циклом активів, призначати їх співробітникам та обробляти заявки на нове обладнання або ремонт.

## Основні можливості

### Backend (для менеджерів)

- **Облік IT-активів** з детальними характеристиками:
  - Інвентарний номер (автогенерація)
  - Категорія (з підтримкою ієрархії)
  - Технічні характеристики
  - Серійний номер, виробник, модель
  - Фінансова інформація (дата і ціна придбання, гарантія)

- **Життєвий цикл активів**:
  - Придбання
  - Доступний
  - Призначений
  - У використанні
  - На ремонті
  - Списаний

- **Управління співробітниками**:
  - Призначення активів співробітникам
  - Історія всіх переміщень
  - Автоматичне відстеження змін

- **Система заявок** з workflow:
  - Створення заявок трьох типів: новий актив, ремонт, заміна
  - Статуси: чернетка → очікування → в роботі → готово
  - Пріоритети (низький, середній, високий, критичний)
  - Призначення відповідальних
  - Коментарі та історія

- **Звітність та аналітика**:
  - Фільтрація за категоріями, статусами, співробітниками
  - Групування даних
  - Kanban-представлення для заявок
  - Статистика по активах

### Frontend (портал для співробітників)

- **Особистий кабінет**:
  - Перегляд закріплених активів
  - Детальна інформація про кожен актив
  - Історія переміщень

- **Система заявок**:
  - Створення заявок на новий актив
  - Заявки на ремонт існуючого обладнання
  - Заявки на заміну
  - Відстеження статусу заявок
  - Історія всіх заявок

## Технічні деталі

### Моделі

#### it.asset
Головна модель для активів з полями:
- Базова інформація (назва, код, категорія, опис)
- Технічні характеристики
- Фінансові дані
- Статус життєвого циклу
- Зв'язок з співробітником
- Історія переміщень
- Пов'язані заявки

Наслідує: `mail.thread`, `mail.activity.mixin` для повної підтримки відстеження змін та активностей.

#### it.asset.category
Категорії з підтримкою ієрархії (parent-child).

#### it.asset.movement
Історія переміщень активів між співробітниками.

#### it.asset.request
Заявки з workflow та інтеграцією з порталом.
Наслідує: `mail.thread`, `mail.activity.mixin`, `portal.mixin`.

### Права доступу

**Дві групи користувачів:**
- **Користувач (User)**: може переглядати активи та створювати заявки
- **Менеджер (Manager)**: повний доступ до всіх функцій

**Правила доступу (Record Rules):**
- Портальні користувачі бачать тільки свої активи та заявки
- Внутрішні користувачі бачать свої заявки
- Менеджери бачать все

### HTTP Контролери

Реалізовані маршрути:
- `/my/assets` - список активів співробітника
- `/my/asset/<id>` - детальна інформація про актив
- `/my/asset-requests` - список заявок
- `/my/asset-requests/<id>` - детальна інформація про заявку
- `/my/asset-requests/new` - форма створення заявки
- `/my/asset-requests/create` - обробка створення заявки

## Встановлення

1. Скопіюйте папку `it_asset_management` в директорію `addons` вашої інсталяції Odoo

2. Перезапустіть Odoo сервер

3. Активуйте режим розробника (Developer Mode)

4. Оновіть список додатків (Apps → Update Apps List)

5. Знайдіть "IT Asset Management" та встановіть модуль

## Початок роботи

### Для менеджерів:

1. **Створіть категорії** (IT Активи → Активи → Категорії)
   - Комп'ютери, Сервери, Мобільні пристрої тощо

2. **Додайте активи** (IT Активи → Активи → Всі активи)
   - Заповніть всі поля
   - Призначте співробітнику (опціонально)

3. **Обробляйте заявки** (IT Активи → Заявки)
   - Переглядайте нові заявки
   - Призначайте відповідальних
   - Змінюйте статуси

### Для співробітників (через портал):

1. Увійдіть на портал під своїм обліковим записом

2. Перейдіть до "Мої активи" для перегляду закріпленого обладнання

3. Створіть заявку через "Мої заявки" → "Створити нову заявку"

4. Відстежуйте статус своїх заявок

## Демонстраційні дані

Модуль включає демо-дані:
- 8 категорій активів
- 9 різних активів (ноутбуки, сервери, телефони тощо)
- 5 прикладів заявок в різних статусах
- Історія переміщень

## Основні функції коду

### Автоматична генерація номерів
```python
@api.model
def create(self, vals):
    if vals.get('code', _('New')) == _('New'):
        vals['code'] = self.env['ir.sequence'].next_by_code('it.asset')
    return super(ITAsset, self).create(vals)
```

### Відстеження змін
Використовується `tracking=True` на важливих полях для автоматичного логування змін.

### Workflow заявок
Методи `action_submit()`, `action_start_progress()`, `action_complete()` тощо для управління статусами.

### Порталний доступ
Контролер `ITAssetPortal` розширює `CustomerPortal` для додавання нових маршрутів.
